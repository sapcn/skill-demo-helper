# Skill 试用演示助手

## 激活时机

当用户说以下内容时激活此 skill：
- "帮我试试这个 skill"
- "演示一下这个 skill"
- "看看这个 skill 能做什么"
- "解读并试用 xxx skill"
- 或者刚用 `npx skills add` 安装了一个新 skill 后

## 工作流程

### 第一步：定位 Skill 目录

1. 首先确定要演示的 skill 名称
2. 查找该 skill 的安装位置，通常在以下路径：
   - 用户全局安装：`%USERPROFILE%\.claude\skills\<skill-name>\SKILL.md`
   - 项目本地安装：`<project-dir>\.claude\skills\<skill-name>\SKILL.md`
3. 读取该 skill 的 `SKILL.md` 文件

### 第二步：解读 Skill

向用户展示以下分析：

1. **Skill 基本信息**
   - 名称和描述
   - 作者和来源

2. **核心功能**
   - 这个 skill 主要解决什么问题
   - 适用于什么场景

3. **使用方法**
   - 如何激活这个 skill
   - 典型的使用命令或触发词

4. **能力边界**
   - 这个 skill 能做什么
   - 不能做什么

### 第三步：生成演示成果

根据 skill 的类型，自动生成一个**可运行的演示**：

| Skill 类型 | 演示方式 |
|-----------|----------|
| 代码生成类 | 生成一个简单的示例项目/代码 |
| 视频/媒体类 | 生成一个最小可运行的示例 |
| 文档/写作类 | 生成一段样本文档 |
| API/工具类 | 调用该工具并展示结果 |
| 工作流类 | 走一遍完整流程 |

**演示原则：**
- 保持简单，突出核心价值
- 代码要能直接运行
- 结果要可视化/可感知
- 避免复杂配置

### 第四步：总结建议

给用户一个清晰的决策建议：

```
┌─────────────────────────────────────────────┐
│  Skill 试用报告                              │
├─────────────────────────────────────────────┤
│  名称: xxx                                   │
│  用途: xxx                                   │
├─────────────────────────────────────────────┤
│  优点:                                       │
│  - xxx                                       │
│  - xxx                                       │
├─────────────────────────────────────────────┤
│  缺点/限制:                                  │
│  - xxx                                       │
├─────────────────────────────────────────────┤
│  建议: [保留/卸载/深入研究]                   │
└─────────────────────────────────────────────┘
```

## 注意事项

- 如果 skill 依赖外部服务，先确认用户是否有相关账号/API key
- 如果需要安装依赖，明确告知用户
- 演示失败时，分析原因并给出解决方案
- 始终用中文与用户交流

## 示例对话

**用户**: 帮我试试刚安装的 remotion-best-practices skill

**助手执行**:
1. 读取 `~/.claude/skills/remotion-best-practices/SKILL.md`
2. 解读：这是一个关于 Remotion 视频框架的最佳实践指南
3. 生成演示：创建一个最简单的 Remotion 视频组件
4. 展示报告和建议
